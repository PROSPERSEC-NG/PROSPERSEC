# OSI Model - Day 1

Summary

This note covers the OSI model layers, purpose, and common protocols seen at each layer.

Layers
1. Physical — bits on the wire (cables, hubs, repeaters)
2. Data Link — MAC, switches, frames (Ethernet, ARP)
3. Network — routing & IP (IPv4/IPv6, ICMP)
4. Transport — end-to-end services (TCP, UDP)
5. Session — sessions, connections management
6. Presentation — encoding, encryption, compression
7. Application — high-level protocols (HTTP, SMTP, DNS)

Quick notes
- OSI is a conceptual model; TCP/IP is the dominant stack in practice.
- Useful for troubleshooting: map observed behavior to layer(s).
